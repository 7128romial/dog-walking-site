📖 סיפורי הצלחה — איך זה עובד טכנית

  • מבנה: 6 כרטיסי סיפור ב-grid של Bootstrap (col-lg-4 col-md-6 col-sm-12) — 3 בשורה במחשב, 2 בטאבלט, 1 בנייד.
  • סינון לפי קטגוריה: לכל כפתור data-filter, לכל כרטיס data-category. בלחיצה — משווים ביניהם ומציגים/מסתירים עם .show() / .hide().
  • 5 קטגוריות: הכל / אימוץ / שיקום / חינוך / חברות. הכפתור הנבחר מקבל class="active".
  • מונה: $(".story-tile:visible").length סופר כמה סיפורים גלויים ומעדכן את #storiesCounter.
  • פתיחת סיפור: לחיצה על כרטיס ← קוראים data-story ← שולפים מאובייקט stories ב-JS ← .text() לכותרת ולתוכן ← slideDown + גלילה לפאנל.
  • סגירה: כפתור "סגור" ← slideUp והסרת ההדגשה מהכרטיס.

❓ שאלות צפויות — סיפורי הצלחה

איך הסינון עובד בלי לרענן דף?jQuery משווה את data-filter של הכפתור ל-data-category של הכרטיס, ומסתיר/מציג ב-DOM — הכל בצד לקוח.
למה .parent() בסינון?כל כרטיס יושב בתוך עמודת Bootstrap (div col). מסתירים את העמודה כדי שה-grid לא יישבר.
למה .text() ולא .html()?text() מכניס טקסט בלבד — בטוח יותר, מונע הרצת קוד/HTML לא רצוי.
מאיפה מגיע תוכן הסיפורים?מאובייקט JavaScript קבוע בקובץ stories.js — לא ממסד נתונים.
איך עובד המונה?סופר את הכרטיסים עם הסלקטור :visible ומציג את המספר אחרי כל סינון.

🏢 אודות — איך זה עובד טכנית

  • סקציות העמוד: Hero, הסיפור שלנו, הערכים, הצוות, צור קשר + מפה, קריאה לפעולה — בנויות עם grid של Bootstrap.
  • כרטיסי צוות: פרטי כל איש צוות מאוחסנים ב-data-name, data-role, data-bio בתוך ה-HTML.
  • פתיחת ביוגרפיה: לחיצה ← קוראים את ה-data-* עם .data() ← כותבים לפאנל עם .html() (כי יש <strong> ו-<br>) ← slideDown.
  • Toggle: לחיצה שנייה על אותו כרטיס סוגרת את הפאנל.
  • נגישות: תמיכה במקלדת — אירוע keypress, מקש Enter (which === 13); tabindex="0", role="button", aria-label, aria-live="polite", ו-skip-link.
  • אפקט fade-in: בגלילה משווים את offset().top של אלמנט לגובה החלון, ומוסיפים class="visible".
  • מפה: Google Maps משובץ ב-<iframe> עם loading="lazy" (טעינה רק כשמגיעים אליו).

❓ שאלות צפויות — אודות

מה זה data attributes?תכונות data-* מותאמות אישית ב-HTML לאחסון מידע. קוראים אותן ב-jQuery עם .data().
למה כאן .html() ובסיפורים .text()?בביוגרפיה מזריקים תגיות (strong, br) לעיצוב, לכן צריך html(). התוכן קבוע ושלי — לכן בטוח.
איך תמכת בנגישות?ניווט במקלדת (Enter), role ו-aria-label לקוראי מסך, aria-live לפאנל, ו-skip-link לדילוג לתוכן.
איך עובד ה-fade-in בגלילה?אירוע scroll בודק אם האלמנט נכנס לתוך המסך, ומוסיף class שמפעיל אנימציית CSS.
למה loading="lazy" במפה?ה-iframe נטען רק כשגוללים אליו — משפר את מהירות טעינת הדף.
למה Bootstrap לפריסה?מערכת grid ו-breakpoints מוכנים לרספונסיביות, וגרסת RTL לתמיכה בעברית.

⚠️ נקודות לשים לב

  • ⚠️ הבדל מרכזי: בסיפורים משתמשים ב-.text(), באודות ב-.html() — דעי להסביר למה (תגיות עיצוב מול טקסט בלבד).
  • ⚠️ שני העמודים — כל הנתונים בצד לקוח (JS / data attributes), ללא מסד נתונים.
  • ⚠️ אם נשאלת על נגישות — זה החוזק של עמוד "אודות": מקלדת + ARIA.
